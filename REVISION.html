1. Architecture générale

Trois classes centrales et un pattern de déplacement de cartes via moveTo. Tout passe par là.

Game
Gère la partie, les piles de réserve (supplyPiles), le tour courant, le trash. Point d'entrée : run().
Player
Un joueur avec 7 listes de cartes + 3 compteurs (money, numberOfActions, numberOfBuys).
Card (abstract)
Chaque carte hérite et redéfinit play(Player p) et/ou getVictoryValue().

Les 7 emplacements de cartes d'un joueur

ListeDescription
handMain du joueur
discardDéfausse
drawPioche (sommet = getLast())
inPlayCartes jouées ce tour
cardsSetAsideMises de côté (Haven, Blockade...)
islandMatTapis Island
nativeVillageMatTapis Native Village

2. La classe Card

Une carte connaît son emplacement actuel via private List<Card> location. La méthode moveTo retire de l'ancienne liste, ajoute à la nouvelle, et met à jour location.

public void moveTo(List<Card> newLocation) {
    if (location != null) {
        location.remove(this);
    }
    location = newLocation;
    newLocation.add(this);
}
⚠️ Règle d'or
Toutes les manipulations de cartes passent par moveTo. Jamais de list.add(card) ou list.remove(card) direct.

4. La classe Game

Méthodes utiles

  • getAvailableSupplyCards() — toutes les cartes au sommet des piles non-vides
  • getCardFromSupply(name) — la carte d'une pile par son nom
  • moveCardToTrash(c) — écarter (trash) une carte (visibilité package, à exposer si besoin)
  • isFinished() — vraie si la pile Province est vide OU si ≥3 piles sont vides
  • moveToNextPlayer() — passe au joueur suivant

À ajouter probablement

public List<Player> getOtherPlayers(Player p) {
    List<Player> others = new ArrayList<>();
    int start = players.indexOf(p);
    for (int i = 1; i < players.size(); i++) {
        others.add(players.get((start + i) % players.size()));
    }
    return others;
}

Ordre du tour pour les attaques : on commence par le joueur juste après le joueur courant.

5. Pattern : Carte Action simple

La majorité des cartes ★ tiennent en quelques lignes. Toujours dans play(Player p) :

@Override
public void play(Player p) {
    p.drawToHand();           // +X cartes (autant de fois que nécessaire)
    p.incrementActions(N);    // +N actions
    p.incrementMoney(N);      // +N pièces
    p.incrementBuys(N);       // +N achats
}

Cartes concernées : Bazaar, Fishing Village, Merchant Ship (avec mémo Duration), Wharf, Caravan, Astrolabe, Sea Chart...

6. Pattern : faire choisir le joueur

NE JAMAIS faire la lecture clavier toi-même. Utilise une des 4 méthodes utilitaires de Player :

MéthodeQuand l'utiliser
chooseCardFromHand(instr, predicate, canPass)Choisir une carte de la main avec filtre
chooseCardFromSupply(instr, predicate, canPass)Choisir une carte de la réserve avec filtre
chooseCardFromButtons(instr, listeCartes, canPass)Choisir parmi des cartes hors main/réserve (ex: cartes révélées)
chooseStringFromButtons(instr, buttons, canPass)Choisir une option (y/n, trash/discard...)

Exemple — Choisir une Action en main

Card c = p.chooseCardFromHand(
    "Choose an Action card",
    card -> card.hasType(CardType.ACTION),
    false  // ne peut pas passer s'il a une Action
);
if (c != null) {
    // jouer la carte choisie
}

Exemple — Choix Yes/No

List<Button> buttons = new ArrayList<>();
buttons.add(new Button("Yes", "y"));
buttons.add(new Button("No", "n"));
String choice = p.chooseStringFromButtons("Play this card?", buttons, false);
if ("y".equals(choice)) { /* ... */ }

7. Pattern : défausser N cartes

int n = 3;
while (!p.getCardsInHand().isEmpty() && n > 0) {
    Card c = p.chooseCardFromHand(
        "Discard %d card(s)".formatted(n),
        false);
    p.moveToDiscard(c);
    n--;
}

Réutilisé pour : Warehouse (3), Sea Witch (2), Tide Pools (2), Cellar...

8. Pattern : attaquer tous les adversaires

for (Player adv : p.getGame().getOtherPlayers(p)) {
    if (isProtected(adv)) continue;   // Lighthouse, Moat...
    // effet d'attaque sur adv
}

Cartes concernées : Cutpurse, Ghost Ship, Sea Hag, Sea Witch, Pirate Ship, Corsair.

9. Pattern : gagner une carte

🚨 Critique
Toujours utiliser p.gainTo(card, location), jamais card.moveTo(list) directement quand on gagne une carte. Pourquoi ? Plein de cartes ont des effets déclenchés par le gain : Sailor, Pirate, Embargo (ajoute des Curse), etc. 
// Achat / gain depuis la réserve
Card silver = p.getCardFromSupply("Silver");
p.gainTo(silver, p.getCardsInDiscard()); // ou autre destination

10. Pattern : itérer sur une copie

⚠️ Bug très fréquent
Si tu déplaces des cartes pendant la boucle, itère sur une copie. Sinon : ConcurrentModificationException ou cartes oubliées. 
// ✅ OK — getCardsInHand() renvoie une copie
for (Card c : p.getCardsInHand()) {
    c.moveTo(discard);
}

// ❌ BUG — la liste est modifiée pendant l'itération
for (Card c : hand) {
    c.moveTo(discard);
}

// ✅ Alternative manuelle
for (Card c : new ArrayList<>(hand)) {
    c.moveTo(discard);
}

11. Exemples de cartes

★ Facile Bazaar

+1 Carte, +2 Actions, +1 Pièce. Aucune interaction.

public class Bazaar extends Card {
    public Bazaar() {
        super("Bazaar", 5, CardType.ACTION);
    }

    @Override
    public void play(Player p) {
        p.drawToHand();
        p.incrementActions(2);
        p.incrementMoney(1);
    }
}

★ Facile Warehouse

+3 Cartes, +1 Action, défausser 3 cartes.

public class Warehouse extends Card {
    public Warehouse() { super("Warehouse", 3, CardType.ACTION); }

    @Override
    public void play(Player p) {
        for (int i = 0; i < 3; i++) p.drawToHand();
        p.incrementActions(1);

        int n = 3;
        while (!p.getCardsInHand().isEmpty() && n > 0) {
            Card c = p.chooseCardFromHand(
                "Discard %d card(s)".formatted(n), false);
            p.moveToDiscard(c);
            n--;
        }
    }
}

★ Facile Cutpurse — attaque automatique

+2 Pièces. Chaque adversaire défausse un Copper de sa main (s'il en a).

@Override
public void play(Player p) {
    p.incrementMoney(2);

    for (Player adv : p.getGame().getOtherPlayers(p)) {
        if (isProtected(adv)) continue; // Lighthouse

        Card copper = null;
        for (Card c : adv.getCardsInHand()) {
            if (c.hasName("Copper")) { copper = c; break; }
        }
        if (copper != null) {
            adv.moveToDiscard(copper);
        }
        // sinon : (optionnel) on log la main de l'adversaire
    }
}

★ Facile Salvager — trash for benefit

+1 Achat. Écarter une carte de la main, +autant de pièces que son coût.

@Override
public void play(Player p) {
    p.incrementBuys(1);

    Card c = p.chooseCardFromHand("Trash a card", false);
    if (c != null) {
        p.incrementMoney(c.getCost());
        p.getGame().moveCardToTrash(c); // exposer en public si besoin
    }
}

★ Facile Island — utilise islandMat

Place cette carte + une carte de la main sur le tapis Island (qui compte comme cartes possédées).

@Override
public void play(Player p) {
    // déplacer Island elle-même sur le tapis
    p.moveToIslandMat(this);

    Card c = p.chooseCardFromHand("Set aside a card on the Island mat", true);
    if (c != null) {
        p.moveToIslandMat(c);
    }
}

Tu auras à ajouter moveToIslandMat(Card c) dans Player qui fait c.moveTo(islandMat).

★★★ Difficile Embargo

+2 Pièces, écarter Embargo, choisir une pile de la réserve qui reçoit un jeton. Quand on achète dans cette pile : autant de Curse que de jetons.

@Override
public void play(Player p) {
    p.incrementMoney(2);
    p.getGame().moveCardToTrash(this);

    Card chosen = p.chooseCardFromSupply(
        "Place an Embargo token on a supply pile",
        c -> true,
        false);
    if (chosen != null) {
        p.getGame().addEmbargoToken(chosen.getName());
    }
}

Dans Game, ajouter Map<String, Integer> embargoTokens. Dans Player.playTurn(), à l'achat : si la pile a des jetons, le joueur gagne autant de Curse.

★ Lookout — chooseCardFromButtons

Révèle 3 cartes du dessus de la pioche : 1 trash, 1 discard, 1 retour sur la pioche.

@Override
public void play(Player p) {
    p.incrementActions(1);

    // Révéler 3 cartes (en gérant les cas où il y en a moins)
    List<Card> revealed = new ArrayList<>();
    for (int i = 0; i < 3; i++) {
        Card c = p.getCardFromDeck();
        if (c == null) break;
        c.moveTo(p.getCardsSetAside()); // ou liste temporaire
        revealed.add(c);
    }

    // 1. Trash
    Card toTrash = p.chooseCardFromButtons(
        "Choose a card to trash", revealed, false);
    if (toTrash != null) {
        revealed.remove(toTrash);
        p.getGame().moveCardToTrash(toTrash);
    }

    // 2. Discard
    Card toDiscard = p.chooseCardFromButtons(
        "Choose a card to discard", revealed, false);
    if (toDiscard != null) {
        revealed.remove(toDiscard);
        p.moveToDiscard(toDiscard);
    }

    // 3. La dernière retourne sur la pioche
    for (Card c : revealed) {
        p.putOnDeck(c); // helper à créer : c.moveTo(draw)
    }
}

Duration Caravan

+1 Carte, +1 Action. Au début de votre prochain tour, +1 Carte.

public class Caravan extends Card {
    public Caravan() {
        super("Caravan", 4, CardType.ACTION, CardType.DURATION);
    }

    @Override
    public void play(Player p) {
        p.drawToHand();
        p.incrementActions(1);

        // Effet retardé : enregistrer un effet à exécuter au prochain tour
        p.addNextTurnEffect(() -> p.drawToHand());
        // (la carte reste en inPlay, ne sera PAS défaussée au cleanup
        //  tant qu'elle a un effet en attente)
    }
}

Tu auras besoin d'un mécanisme central. Voir section Duration ci-dessous.

Reaction Lighthouse

+1 Action, +1 Pièce maintenant. Au prochain tour : +1 Pièce. Tant qu'il est en jeu, le joueur est immunisé contre les attaques.

@Override
public void play(Player p) {
    p.incrementActions(1);
    p.incrementMoney(1);
    p.addNextTurnEffect(() -> p.incrementMoney(1));
}

Pour la protection, dans chaque carte ATTACK avant d'attaquer :

private boolean isProtected(Player adv) {
    for (Card c : adv.getCardsInPlay()) {
        if (c.hasName("Lighthouse")) return true;
    }
    return false;
}

12. Cartes Duration (le sujet le plus complexe)

Les cartes Duration ont un effet qui s'étale sur 2 tours :

  1. Tour T : effet immédiat + carte reste en inPlay (NON défaussée).
  2. Tour T+1 : effet "au début du prochain tour" + défausse à la fin du tour T+1.

Approche recommandée — file d'effets différés

Dans Player, ajouter une liste List<Runnable> nextTurnEffects :

private final List<Runnable> nextTurnEffects = new ArrayList<>();

public void addNextTurnEffect(Runnable effect) {
    nextTurnEffects.add(effect);
}

// au début de playTurn(), AVANT d'initialiser les compteurs :
public void playTurn() {
    List<Runnable> toRun = new ArrayList<>(nextTurnEffects);
    nextTurnEffects.clear();
    for (Runnable r : toRun) r.run();

    numberOfActions = 1;
    money = 0;
    numberOfBuys = 1;
    // ... reste du code
}

Modifier cleanup() pour ne PAS défausser les Duration actives

public void cleanup() {
    numberOfActions = 0;
    money = 0;
    numberOfBuys = 0;

    for (Card c : getCardsInHand()) c.moveTo(discard);

    for (Card c : getCardsInPlay()) {
        // Une carte Duration reste en jeu si elle a encore un effet
        if (c.hasType(CardType.DURATION) && c.hasPendingEffect()) {
            // on la laisse en inPlay
        } else {
            c.moveTo(discard);
        }
    }

    for (int i = 0; i < 5; i++) drawToHand();
}

Tu peux aussi gérer ça avec un attribut booléen pendingEffect sur Card, mis à true quand on enregistre un effet retardé, false après exécution.

14. Protection contre les attaques

Lighthouse : tant qu'il est en jeu, son joueur est immunisé contre les attaques. Avant chaque effet d'attaque sur un adversaire, vérifier :

private boolean isProtected(Player adv) {
    return adv.getCardsInPlay().stream()
        .anyMatch(c -> c.hasName("Lighthouse"));
}

public int getVictoryPoints() {

18. Pièges fréquents

🪤 Piège #1 — moveTo direct au lieu de gainTo
Si tu fais card.moveTo(p.discard) au lieu de p.gainTo(card, ...), les triggers sur gain (Pirate, Sailor, Embargo→Curse) ne se déclencheront pas.
🪤 Piège #2 — itérer sur la liste originale
for (Card c : hand) + c.moveTo(discard) = bug. Toujours getCardsInHand().
🪤 Piège #3 — Duration défaussée trop tôt
Si tu défausses systématiquement les Duration au cleanup du tour T, leur effet du tour T+1 ne s'exécutera pas.
🪤 Piège #4 — getCardFromDeck() ne retire pas la carte
Cette méthode renvoie la carte du sommet sans la retirer. Pour la mettre en main, utilise drawToHand() (qui fait le moveTo(hand)).
🪤 Piège #5 — Embargo sur achat seulement
Le malus Curse de l'Embargo ne se déclenche que sur un achat (pendant la boucle d'achat de playTurn), pas sur un gain provoqué par une autre carte.
🪤 Piège #6 — Smugglers
Mémoriser les cartes gagnées par chaque joueur lors de son dernier tour (pas tous les tours). À mettre à jour au début ou à la fin de chaque tour.

Pour chaque carte, lis d'abord sa section dans IO.md — il y a tous les choix attendus, les cas limites et les remarques importantes.
