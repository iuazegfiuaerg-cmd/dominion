Presentation du projet

Dominion Seaside est un jeu de cartes de type deck-building. A tour de role, les joueurs jouent des cartes pour gagner de l'argent, des actions, et acheter de nouvelles cartes. Le gagnant est celui qui a le plus de Points de Victoire a la fin.

L'extension Seaside introduit notamment les cartes Duration dont l'effet persiste sur le tour suivant.

Objectif : implementer en Java les cartes Kingdom de l'extension Seaside (35 cartes) ainsi que les cartes communes (Tresor, Victoire, Malediction). Le code est evalue par des tests unitaires automatises.

Architecture des classes

3 paquetages : fr.umontpellier.iut.dominion, .cards, .cards.seaside / .common.

Card (abstraite)

Classe mere de toutes les cartes. Attributs : name, cost, types, location.

Methodes cles : play(Player), getVictoryValue(), hasDurationEffect(), atStartOfTurn(Player), moveTo(List), hasType(CardType).

CardType (enum)

7 types : TREASURE, ACTION, VICTORY, CURSE, REACTION, ATTACK, DURATION. Une carte peut combiner plusieurs types.

Player

7 emplacements : hand, discard, draw, inPlay, cardsSetAside, islandMat, nativeVillageMat.

Compteurs : money, numberOfActions, numberOfBuys.

Game

Gere la liste des joueurs, les piles de reserve (supplyPiles), les cartes ecartees (trashedCards) et le deroulement de la partie.

SupplyPile

Pile de reserve pour un type de carte. Etend ArrayList<Card>. Possede un compteur nbEmbargoTokens.

FactorySupplyPile

Fabrique qui cree une SupplyPile a partir du nom d'une carte (Factory Pattern).

Mecaniques cles a retenir

Tour d'un joueur (playTurn())

  1. Initialise numberOfActions = 1, numberOfBuys = 1, money = 0.
  2. Appelle atStartOfTurn(p) pour chaque carte en jeu (effets Duration).
  3. Le joueur peut jouer des Actions, puis des Tresors, puis acheter des cartes.
  4. Quand un Tresor est joue, plus aucune Action n'est possible.

Cleanup (cleanup())

  • Toutes les cartes en main sont defaussees.
  • Les cartes en jeu sont defaussees sauf si hasDurationEffect() renvoie true.
  • Treasury : peut etre placee sur la pioche si aucune carte Victoire n'a ete recue.
  • Pioche 5 cartes (3 si hasOutpostEffect).

Cartes Duration

Une carte Duration reste en jeu tant que hasDurationEffect() retourne true. L'effet est applique via atStartOfTurn(Player) au debut du tour suivant. Le pattern habituel :

private boolean durationActive;

public void play(Player p) {
    // effet immediat
    durationActive = true;
}
public void atStartOfTurn(Player p) {
    // effet du prochain tour
    durationActive = false;
}
public boolean hasDurationEffect() { return durationActive; }

Cartes Attack et protection

Une carte Lighthouse en jeu protege son proprietaire : isProtectedFromAttacks() renvoie true. Toutes les cartes Attaque doivent verifier ce flag avant de toucher un adversaire.

Hooks dans Player.gainTo()

Quand un joueur gagne une carte, plusieurs reactions sont declenchees :

  • Embargo : si la pile a des jetons, le joueur recoit autant de Maledictions.
  • Pirate : si un Tresor est gagne, chaque joueur peut jouer son Pirate.
  • Blockade : si la carte gagnee est celle mise de cote, l'adversaire recoit une Malediction.
  • Monkey : si le voisin de droite gagne une carte, le proprietaire pioche +1.
  • Sailor : si une carte Duration est gagnee, le joueur peut la jouer (1 fois/tour).

Fichiers principaux modifies

Pour faire fonctionner les cartes Seaside, le squelette de Player.java, Card.java et SupplyPile.java a du etre etendu avec de nouvelles methodes et attributs.

Card.java

  • hasDurationEffect() : retourne false par defaut, redefinie par chaque carte Duration.
  • atStartOfTurn(Player) : vide par defaut, redefinie pour executer l'effet du prochain tour.

Player.java - nouveaux attributs

  • hasOutpostEffect, isOutpostTurn (Outpost)
  • cardsGainedThisTurn, cardsGainedLastTurn (Smugglers)
  • gainedVictoryCardThisTurn (Treasury)
  • cardsSetAside, islandMat, nativeVillageMat

Player.java - methodes ajoutees

  • isProtectedFromAttacks() : verifie si Lighthouse est en jeu.
  • gainTo(Card, List) : centralise tous les hooks (Embargo, Pirate, Blockade, Monkey, Sailor).
  • moveToHand/Discard/Draw/SetAside/Trash(Card) : deplacements de cartes.
  • moveToNativeVillageMat(Card), takeAllFromNativeVillageMat()
  • addIslandCard(Card)
  • getBottomCardFromDeck(), moveBottomCardToDeckTop() (Pearl Diver)
  • gainCardToHand(String), gainToHand(Card), gainToDiscard(Card)
  • Modification de playCard() pour Corsair, et cleanup() pour Treasury et Duration.

SupplyPile.java

  • nbEmbargoTokens, addEmbargoToken(), getNbEmbargoTokens().

Game.java

  • getOtherPlayers(Player), getSupplyPile(String), moveCardToTrash(Card).
